# 多模态大模型对话助手 (MiniCPM-V) UI/UX 结构梳理

## 1. 整体架构概述
该 App 是一个支持**图文混合输入**、**流式文本输出**的多轮对话系统。整体布局采用标准的现代聊天应用结构：顶部导航栏 + 中间可滚动的内容区 + 底部固定输入区。

## 2. 界面区域拆解 (自上而下)

### 2.1 顶部导航栏 (Top App Bar)
*   **左侧:** App Logo 图标与应用名称 ("MiniCPM-V")。
*   **中间:** 当前加载的模型名称 (图示为 "MiniCPM-V")。
*   **右侧操作区:**
    *   **清空按钮 (Trash Icon):** 触发事件：清空当前对话历史和上下文缓存。
    *   **设置按钮 (Gear Icon):** 触发事件：跳转至高级设置页（如系统提示词配置、推理参数调整等，图中未展示）。

### 2.2 主内容区 (Main Content Area) - 包含两种状态

**状态 A：空状态 / 欢迎页 (对应图 1)**
*   **欢迎文案:** 居中显示的大标题 ("Welcome to MiniCPM-V") 及副标题说明。
*   **快捷指令提示卡片 (Suggestion Chips):** 横排滑动的卡片，如“请描述图片中的内容”。点击事件：将卡片文本自动填入底部输入框，或直接触发发送。
*   **【核心组件】图片待发送预览区:**
    *   **图像展示:** 居中显示用户刚选中的图片。
    *   **元数据信息:** 底部显示图像分辨率 (如 448x448)、文件大小 (如 314 KB)、预处理耗时 (如 0.3s)。
    *   **状态条:** 绿色的进度条，表示图像已被底层引擎（如 CLIP）成功预处理，准备好进行多模态推理。

**状态 B：对话状态 (对应图 2)**
*   **消息列表 (LazyColumn):** 垂直滚动的对话流。
*   **图片气泡 (顶部):** 如果是图文对话，当前轮次的图片会作为独立区块显示在最上方（带有绿色的预处理状态条）。
*   **用户消息气泡 (User Bubble):** 靠右对齐，包含用户输入的文本（如“请描述图片中的内容。”）。背景色区别于 AI 气泡。
*   **AI 消息气泡 (AI Bubble):** 靠左对齐，背景色通常为浅灰色。
    *   **【关键特性】Markdown 渲染:** 必须支持解析和渲染 Markdown 格式（如 `### 标题`、`**加粗**`、`1. 2. 列表`）。
    *   **【关键特性】打字机效果:** 支持流式 (Streaming) 接收到底层 Token 时的逐字更新显示。

### 2.3 悬浮操作区 (Floating Action Area)
*   **终止生成按钮 (Stop Button):** 
    *   **显示逻辑:** 仅在 AI 正在生成回复（Streaming 状态）时悬浮显示在输入区正上方。
    *   **触发事件:** 中断底层的推理线程，停止接收新的 Token。

### 2.4 底部输入区 (Bottom Input Area)
固定在屏幕底部的操作栏，包含三个核心元素：
1.  **文本输入框 (TextField):** 提示词“发消息”。支持多行输入。
2.  **选择图片按钮 (Image Picker Icon):** 
    *   触发事件：调用系统相册/文件选择器获取图片 URI。
    *   业务逻辑：选中图片后，触发 UI 进入“状态 A”的图片预览模式，并在后台触发 C++ 层的图像特征提取 (Image Encoding)。
3.  **发送按钮 (Send Icon):**
    *   状态逻辑：无文本且无图片时置灰不可点击；有文本或有准备好的图片时高亮（蓝色）。
    *   触发事件：将当前的图片特征和文本 Prompt 组合，构建 Message 对象加入对话列表，并触发底层大模型进行推理生成。

## 3. 核心业务流程描述 (Workflow for Coder Agent)

### 流程一：图文对话发送流程
1.  **用户点击图片图标：** 调用 Android `ActivityResultContracts.GetContent()` 选择图片。
2.  **UI 更新：** 将选中的图片显示在“图片待发送预览区”。
3.  **后台预处理：** 将图片转为 Bitmap/Byte Array 传入 JNI，由 `llama.cpp` 的 `clip` 模块进行特征提取，UI 显示“预处理耗时”。
4.  **用户输入文本：** 在底部输入框键入 Prompt。
5.  **点击发送：**
    *   清空输入框和预览区。
    *   将包含图片路径和文本的 `UserMessage` 插入 UI 数据模型 (ViewModel)，UI 滚动至最底部显示用户气泡。
    *   插入一个空的 `AIMessage`，进入 Loading/Streaming 状态。
    *   显示“终止生成”悬浮按钮。

### 流程二：流式接收与渲染流程
1.  **JNI 回调：** C++ 端每生成一个 Token，通过 Callback 传回 Kotlin 层。
2.  **状态更新：** ViewModel 接收到 Token，追加到当前的 `AIMessage` 字符串末尾。
3.  **UI 重组：** Jetpack Compose 监听到状态变化，重新渲染该 AI 消息气泡。
4.  **Markdown 解析：** 渲染组件实时对更新后的完整字符串进行 Markdown 解析并展现样式。
5.  **结束状态：** 收到结束符 (EOS)，隐藏“终止生成”按钮，完成单次对话。

## 4. 建议的数据模型 (Data Model)
为了支撑上述 UI，建议定义如下数据类：

```kotlin
enum class Role { USER, ASSISTANT }

data class Message(
    val id: String = UUID.randomUUID().toString(),
    val role: Role,
    val text: String,
    val imageUri: String? = null, // 用户附加的图片路径
    val isStreaming: Boolean = false // 是否正在生成中
)

data class UiState(
    val messages: List<Message> = emptyList(),
    val selectedImageUri: String? = null, // 当前待发送的图片
    val imageProcessStatus: String? = null, // 如 "预处理耗时: 0.3s"
    val isGenerating: Boolean = false // 控制终止按钮的显示
)
```
